# Trello- Criar Card Aprendiz (ID: Zc85wyQAzpZsed9i)\n\nTrigger: Gmail Trigger\n\nVisão Geral:\nEste workflow cria um cartão (card) a partir de e-mails recebidos e notifica o responsável via Gmail. Utiliza um agente de linguagem (LangChain) para tratar prompts, memória simples para contexto por thread e um nó de código para normalizar os campos do lead. Também chama um sub-workflow quando necessário.\n\nPasso a Passo (Node a Node):\n1. Gmail Trigger (Gmail Trigger)\n   - Gatilho principal: monitora a caixa configurada para novos e-mails. Quando chega um e-mail, dispara o fluxo.\n\n2. AI Agent (@n8n/n8n-nodes-langchain.agent)\n   - Recebe o evento do Gmail Trigger e processa o prompt definido. Usa o modelo para entender intenções e extrair informações quando aplicável. O prompt atual pede para avisar um dia antes em casos de e-mail com agenda.\n\n3. OpenAI Chat Model (@n8n/n8n-nodes-langchain.lmChatOpenAi)\n   - Modelo gpt-5-mini configurado como backend do agente para geração de linguagem.\n\n4. Simple Memory (@n8n/n8n-nodes-langchain.memoryBufferWindow)\n   - Armazena contexto por threadId (sessão) para permitir conversas sequenciais com memória curta.\n\n5. HTTP Request (n8n-nodes-base.httpRequestTool)\n   - Utilizado pelo agente para chamadas externas (ex: serviços de vault/segurança). Atualmente apontado para uma URL do Bitwarden.\n\n6. Formaliza dados (n8n-nodes-base.code)\n   - Nó em JavaScript que normaliza a entrada em um objeto lead com campos: nome, email, cidade, pais e idade. Retorna apenas o JSON padronizado.\n\n7. Edit Fields (n8n-nodes-base.set)\n   - Mapeia explicitamente os campos nome, email, cidade, pais e idade para garantir consistência entre nós subsequentes. Mantém outros campos quando presentes.\n\n8. Send a message (n8n-nodes-base.gmail)\n   - Envia um resumo/alerta para giovanni.neves@ciadeestagios.com.br com os campos do lead (nome, email, idade).\n\n9. Call n8n Workflow Tool (@n8n/n8n-nodes-langchain.toolWorkflow)\n   - Possibilidade de acionar um sub-workflow (My Sub-Workflow 1) para lógica adicional (ex: criação do card no Trello).\n\nVisão Técnica / Observações:\n- O fluxo mistura nós de AI (LangChain/OpenAI) com nós tradicionais do n8n (Gmail, HTTP, Code).\n- Credenciais sensíveis (ex: OpenAI, Gmail) são referenciadas por id e devem estar configuradas no ambiente n8n.\n- O nó de código não valida explicitamente email; considerar acrescentar validação/filtragem e tratamento de erros antes do envio.\n\nChangelog:\n- 2026-04-13T14:17:24.169Z — Documentação atualizada.
